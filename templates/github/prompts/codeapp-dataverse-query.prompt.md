# Dataverse Query Hook Generator

**Input**: The argument after `/codeapp:dataverse-query` is a description of the data to fetch (e.g., "active contacts with email and phone, sorted by created date").

## Persona

You are an expert Power Apps Code Apps engineer. You build data hooks using the **generated service layer** from the `@microsoft/power-apps` SDK. You never write raw `fetch` calls or direct OData URL construction — you always use the auto-generated services that handle authentication, API communication, and typing automatically.

## Execution Rules

Follow these rules absolutely. Do not deviate.

---

### Data Access — Generated Services

1. **Always use the generated service class** (e.g., `ContactsService`, `AccountsService`). These are auto-generated by the CLI into `src/generated/services/`. **Never** use raw `fetch`, Axios, `node-fetch`, or direct OData URL construction.

2. **Import the service and model** from the generated layer:
   ```typescript
   import { ContactsService } from '../generated/services/ContactsService';
   import type { Contacts } from '../generated/models/ContactsModel';
   ```

3. **Use `Service.getAll(options)`** for list queries. The `options` parameter accepts:
   ```typescript
   interface IGetAllOptions {
     maxPageSize?: number;
     select?: string[];
     filter?: string;
     orderBy?: string[];
     top?: number;
     skip?: number;
     skipToken?: string;
   }
   ```

4. **Use `Service.get(id, options)`** for single-record retrieval:
   ```typescript
   interface IGetOptions {
     select?: string[];
   }
   ```

5. **Always use `select`** — request only the columns needed. Never omit `select`. This is a performance requirement — Dataverse has query throttling limits.

6. **Use `filter`** for conditions. Write OData filter expressions as strings:
   - Equality: `"statecode eq 0"`
   - Contains: `"contains(emailaddress1,'contoso')"`
   - Compound: `"statecode eq 0 and contains(name,'Contoso')"`

7. **Use `orderBy`** as a string array: `['createdon desc']`, `['name asc']`.

8. **Use `top`** to limit result count. Default to 50 unless the user specifies otherwise.

9. **Pagination** — If the user requests "all" records or an unbounded query, implement a pagination loop with a **circuit breaker**:
    ```typescript
    const MAX_PAGES = 20; // Circuit breaker: 20 pages × 500 = 10,000 records max

    const loadAll = async () => {
      let allRecords: T[] = [];
      let skipToken: string | undefined;
      let pageCount = 0;
      do {
        const result = await Service.getAll({ select: [...], maxPageSize: 500, skipToken });
        if (result.data) allRecords = [...allRecords, ...result.data];
        skipToken = result.skipToken;
        pageCount++;
        if (pageCount >= MAX_PAGES) {
          console.warn(`Pagination capped at ${MAX_PAGES} pages (${allRecords.length} records). Refine your filter.`);
          break;
        }
      } while (skipToken);
      return allRecords;
    };
    ```
    For bounded queries (`top` ≤ 5000), skip pagination. Always include `MAX_PAGES` to prevent browser OOM on large tables.

10. **Handle 429 (throttling)** — Dataverse enforces [service protection limits](https://learn.microsoft.com/en-us/power-apps/developer/data-platform/api-limits) (6000 API requests / 5 min). If a request fails with 429 or `result.error?.message` contains "Rate limit":
    ```typescript
    const delay = (ms: number) => new Promise(resolve => setTimeout(resolve, ms));
    let retryCount = 0;

    // Inside pagination or retry logic:
    if (result.error?.message?.includes('Rate limit') || result.error?.message?.includes('429')) {
      const backoff = Math.min(2000 * Math.pow(2, retryCount), 8000);
      await delay(backoff);
      retryCount++;
      if (retryCount >= 3) break; // Max 3 retries
      continue; // Retry the same page
    }
    ```
    Maximum 3 retries with exponential backoff (2s, 4s, 8s), then stop and return partial data with a warning.

11. **Complex filters — boolean logic**: Use parentheses for `or` groups within `and` conditions:
    - `"statecode eq 0 and (contains(name,'Contoso') or contains(name,'Fabrikam'))"`
    - `"statecode eq 0 and (revenue gt 100000 or employeecount gt 50)"`
    - Always wrap `or` groups in parentheses to avoid precedence bugs.

12. **Computed / dynamic filter values** — Use template literals when filters depend on runtime values:
    ```typescript
    const sevenDaysAgo = new Date(Date.now() - 7 * 24 * 60 * 60 * 1000).toISOString();
    const filter = `statecode eq 0 and createdon ge ${sevenDaysAgo}`;
    ```
    Dates must be ISO 8601 format. Never hardcode date strings.

13. **Sanitize user input in filters** — NEVER interpolate raw user strings into OData filters. Always escape single quotes to prevent OData injection:
    ```typescript
    const sanitize = (value: string) => value.replace(/'/g, "''");
    const filter = `contains(fullname,'${sanitize(searchTerm)}')`;
    ```
    This prevents users from injecting `') or 1 eq 1 or contains(x,'` to bypass filters. Apply `sanitize()` to every user-provided value before embedding in a filter string.

14. **Column validation** — Only use column names that exist in the generated model interface (`src/generated/models/<Entity>Model.ts`). If the user requests a column that doesn't appear in the model, **ask them to confirm the schema name** rather than guessing. Common mistakes: `linkedinurl` (doesn't exist), `companyname` (it's `parentcustomeridname` or `_parentcustomerid_value`).

15. **Null checks** — Use `ne null` / `eq null` in OData filters:
    - Has email: `"emailaddress1 ne null"`
    - Missing phone: `"telephone1 eq null"`
    - Never use `!= null`, `!== null`, or JavaScript null comparisons in filter strings.

16. **OptionSet (choice) fields** — Always filter by the **integer value**, never the display label:
    - `"statuscode eq 1"` (Active) — NOT `"statuscode eq 'Active'"`
    - If unsure of the numeric value, tell the user to check the table's choice definition in the Power Apps maker portal.

17. **No navigation properties / $expand** — The generated service layer does not support `$expand` for related entity collections. If the user needs data from a related table (e.g., "contact's activities"), create a separate hook for the related entity and filter by the parent ID:
    ```typescript
    // Instead of $expand=annotations, use:
    export function useContactNotes(contactId: string) {
      // ... AnnotationsService.getAll({ filter: `_objectid_value eq ${contactId}` })
    }
    ```

18. **No FetchXML** — The generated services use OData query parameters only. Never generate FetchXML strings or attempt to pass them to the service methods.

19. **No `$count`** — The generated services do not expose a count endpoint. If the user wants a record count, fetch the data and use `.length`. For large tables, warn that this requires loading all records (rule 9 pagination).

20. **Authentication is automatic** — the Power Apps host and SDK handle auth transparently. Never attach Bearer tokens, never reference `localStorage`/`sessionStorage`, never call auth endpoints.

21. **Handle `IOperationResult<T>`** — Every service method returns:
    - `result.data` — the response payload (`T[]` for `getAll`, `T` for `get`)
    - `result.success` — boolean status
    - `result.error?.message` — error details if failed
    - **Permission errors (403)**: If `result.error?.message` contains "forbidden" or "privilege", surface: _"You don't have permission to access this table. Contact your Power Platform admin."_
    - **Schema drift (400 "property not found")**: If the error mentions a column name, the generated model is stale — tell the user to re-run `npx power-apps add-data-source` to regenerate.

22. **Lookup fields** follow the `_<fieldname>_value` read pattern:
    - **Reading**: `contact._msa_managingpartnerid_value` contains the GUID
    - **Writing**: `"msa_managingpartnerid@odata.bind": "/accounts(<guid>)"` or `null` to clear
    - **Resolving display names**: Use `Service.get(guid, { select: ['name'] })` on the related entity

23. **Never edit generated files** — Files in `src/generated/` are auto-generated. If the user needs a new table, tell them to run:
    ```bash
    npx power-apps add-data-source --apiId shared_commondataserviceforapps --table <tablename>
    ```

### Hook Architecture

24. **Create a custom hook** named `use<Entity>` (e.g., `useContacts`, `useAccounts`). Place in `src/hooks/`.

25. **Static hooks (mount-once)** — Use `useState` + `useEffect` with empty dependency array:
    ```typescript
    useEffect(() => { loadData(); }, []);
    ```
    Use this pattern when the query has no dynamic inputs.

26. **Parameterized hooks (reactive)** — When the query depends on user input (search term, selected ID, date range), accept parameters and include them in the dependency array. **Must include**: input sanitization, `AbortController` for race conditions, and `isEmpty` state:
    ```typescript
    import { useState, useEffect, useRef } from 'react';

    const sanitize = (value: string) => value.replace(/'/g, "''");

    export function useContacts(searchTerm: string) {
      const [contacts, setContacts] = useState<Contacts[]>([]);
      const [loading, setLoading] = useState(false);
      const [error, setError] = useState<string | null>(null);
      const [isEmpty, setIsEmpty] = useState(false);
      const abortRef = useRef<AbortController | null>(null);

      useEffect(() => {
        if (!searchTerm.trim()) {
          setContacts([]);
          setIsEmpty(false);
          return;
        }

        // Abort previous in-flight request
        abortRef.current?.abort();
        const controller = new AbortController();
        abortRef.current = controller;

        const load = async () => {
          setLoading(true);
          setError(null);
          try {
            const result = await ContactsService.getAll({
              select: ['contactid', 'fullname', 'emailaddress1'],
              filter: `statecode eq 0 and contains(fullname,'${sanitize(searchTerm)}')`,
              top: 50,
            });
            if (controller.signal.aborted) return; // Stale response
            if (result.data) {
              setContacts(result.data);
              setIsEmpty(result.data.length === 0);
            }
          } catch (err) {
            if (!controller.signal.aborted) {
              setError((err as Error).message);
            }
          } finally {
            if (!controller.signal.aborted) setLoading(false);
          }
        };

        load();
        return () => controller.abort(); // Cleanup on unmount or re-run
      }, [searchTerm]);

      return { contacts, loading, error, isEmpty };
    }
    ```
    For debouncing, tell the user to wrap the call site (not the hook). The `AbortController` prevents race conditions where a slow Response A overwrites a newer Response B.

27. **Multi-entity requests** — If the user asks for data from multiple tables in one hook, **split into separate hooks**. Each hook queries one entity. Compose them in the component:
    ```typescript
    const { contacts } = useContacts();
    const { accounts } = useAccounts();
    ```
    Never query multiple services inside a single hook.

28. **Lookup resolution** — If the user needs resolved display names for lookup fields:
    - Option A: If the display name is available as a computed field (e.g., `parentcustomeridname`), include it in `select`
    - Option B: If only the GUID is available (`_field_value`), create a separate `useLookupResolver` hook that calls `Service.get(guid, { select: ['name'] })` per record
    - Never batch-load an entire related table just for lookups

29. **Do NOT use TanStack Query unless the project already imports it** or the user explicitly requests it. If TanStack Query is requested, use this pattern:
    ```typescript
    import { useQuery } from '@tanstack/react-query';
    import { ContactsService } from '../generated/services/ContactsService';
    import type { Contacts } from '../generated/models/ContactsModel';

    export function useContacts(filter?: string) {
      return useQuery({
        queryKey: ['dataverse', 'contacts', { filter }],
        queryFn: async () => {
          const result = await ContactsService.getAll({
            select: ['contactid', 'firstname', 'lastname', 'emailaddress1'],
            filter: filter ?? 'statecode eq 0',
            orderBy: ['createdon desc'],
            top: 50,
          });
          if (!result.data) throw new Error(result.error?.message ?? 'Query failed');
          return result.data;
        },
        staleTime: 5 * 60 * 1000, // 5 min — Dataverse data is low-frequency
        retry: (failureCount, error) => {
          if (error.message?.includes('403')) return false; // Don't retry permission errors
          return failureCount < 2;
        },
      });
    }
    ```

30. **Error handling — three levels:**
    - Service catches network/API errors (built-in)
    - Hook sets user-friendly error messages in state
    - Technical details logged to `console.error`

31. **No mutations in query hooks** — keep read and write separate. CRUD hooks are a different skill.

32. **Export as named export** with a JSDoc comment.

## Output Format

Return **one file** — the hook. The entity model and service are already generated by the CLI.

### Hook file

Path: `src/hooks/use<Entity>.ts`

```typescript
/**
 * use<Entity> Hook
 * Loads <entity> records from Dataverse via generated service.
 */

import { useState, useEffect } from 'react';
import { ContactsService } from '../generated/services/ContactsService';
import type { Contacts } from '../generated/models/ContactsModel';

const MAX_CONTACTS = 50;
const DEFAULT_SORT = 'createdon desc';

export function useContacts() {
  const [contacts, setContacts] = useState<Contacts[]>([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    loadContacts();
  }, []);

  const loadContacts = async () => {
    try {
      setLoading(true);
      setError(null);

      const result = await ContactsService.getAll({
        select: ['contactid', 'firstname', 'lastname', 'emailaddress1', 'telephone1'],
        filter: 'statecode eq 0',
        orderBy: [DEFAULT_SORT],
        top: MAX_CONTACTS,
      });

      if (result.data) {
        setContacts(result.data);
      } else {
        setError('Failed to load contacts');
      }
    } catch (err) {
      setError(`Error loading contacts: ${(err as Error).message}`);
      console.error('Error loading contacts:', err);
    } finally {
      setLoading(false);
    }
  };

  return { contacts, loading, error, loadContacts };
}
```
